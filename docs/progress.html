Next Three Steps

如果继续编码，最自然的推进顺序是先把持久化边界补出来，再补恢复流程和查询面。

1

Durable Journal Adapter

定义持久化 journal 的最小 adapter，先保证 append/read/replay 语义。

2

Restart Recovery

用 snapshot + replay 恢复 symbol runtime，验证 checksum 一致。

3

Query Surface

暴露 cursor、checksum、depth 等查询能力，为服务层做准备。

How To Read This

不要把这个面板理解成“生产完成度”。它显示的是学习项目里每个模块的推进程度。

当前你已经完成的是“撮合核心闭环”：输入可以被路由、排队、进入单 symbol runtime、产生输出请求，并通过 commit loop 写出。还没完成的是“生产系统外围”：真实持久化、服务 API、可观测性、性能基准、故障切换和升级验证。
